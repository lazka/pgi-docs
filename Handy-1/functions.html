Handy.init()¶

Call this function just after initializing GTK, if you are using Gtk.Application it means it must be called when the Gio.Application ::startup signal is emitted. If libhandy has already been initialized, the function will simply return.

This makes sure translations, types, themes, and icons for the Handy library are set up properly.
