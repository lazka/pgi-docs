class ClutterGst.VideoSink(**kwargs)¶
Bases:GstBase.BaseSink, GstVideo.Navigation
Abstract:No
Structure:ClutterGst.VideoSinkClass

Class implementing a GStreamer sink element for Clutter.Texture s.

The ClutterGst.VideoSink structure contains only private data and should not be accessed directly.

static new(texture)¶
Parameters:texture (Clutter.Texture) – a Clutter.Texture
Returns:a Gst.Element for the newly created video sink
Return type:Gst.Element

Creates a new GStreamer video sink which uses texture as the target for sinking a video stream from GStreamer.

This function has to be called from Clutter’s main thread. While GStreamer will spawn threads to do its work, we want all the GL calls to happen in the same thread. Clutter-gst knows which thread it is by assuming this constructor is called from the Clutter thread.

Deprecated since version 1.6: Use Gst.ElementFactory.make (“cluttersink”, ...) and the “texture” GObject.Object property instead.

Property Details¶

ClutterGst.VideoSink.props.texture¶
Name:texture
Type:Clutter.Texture
Default Value:None
Flags:READABLE, WRITABLE

This is the texture the video is decoded into. It can be any Clutter.Texture, however Cluter-Gst has a handy subclass, ClutterGst.VideoTexture, that implements the Clutter.Media interface.

ClutterGst.VideoSink.props.update_priority¶
Name:update-priority
Type:int
Default Value:100
Flags:READABLE, WRITABLE

Clutter-Gst installs a GLib.Source to signal that a new frame is ready to the Clutter thread. This property allows to tweak the priority of the source (Lower value is higher priority).

New in version 1.0.
