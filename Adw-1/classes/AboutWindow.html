Adw.AboutWindowClass

A window showing information about the application.

<picture> <source srcset=”about-window-dark.png” media=”(prefers-color-scheme: dark)”> <img src=”about-window.png” alt=”about-window”> </picture>

An about window is typically opened when the user activates the About … item in the application’s primary menu. All parts of the window are optional.

Main page

AdwAboutWindow prominently displays the application’s icon, name, developer name and version. They can be set with the [property`AboutWindow`:py:data::application-icon<Adw.AboutWindow.props.application_icon>], [property`AboutWindow`:py:data::application-name<Adw.AboutWindow.props.application_name>], [property`AboutWindow`:py:data::developer-name<Adw.AboutWindow.props.developer_name>] and [property`AboutWindow`:py:data::version<Adw.AboutWindow.props.version>] respectively.

What’s New

AdwAboutWindow provides a way for applications to display their release notes, set with the [property`AboutWindow`:py:data::release-notes<Adw.AboutWindow.props.release_notes>] property.

Release notes are formatted the same way as AppStream descriptions.

The supported formatting options are:

  • Paragraph (<p>)

  • Ordered list (<ol>), with list items (<li>)

  • Unordered list (<ul>), with list items (<li>)

Within paragraphs and list items, emphasis (<em>) and inline code (<code>) text styles are supported. The emphasis is rendered in italic, while inline code is shown in a monospaced font.

Any text outside paragraphs or list items is ignored.

Nested lists are not supported.

Only one version can be shown at a time. By default, the displayed version number matches [property`AboutWindow`:py:data::version<Adw.AboutWindow.props.version>]. Use [property`AboutWindow`:py:data::release-notes-version<Adw.AboutWindow.props.release_notes_version>] to override it.

Details

The Details page displays the application comments and links.

The comments can be set with the [property`AboutWindow`:py:data::comments<Adw.AboutWindow.props.comments>] property. Unlike [property`Gtk`.AboutDialog:comments], this string can be long and detailed. It can also contain links and Pango markup.

To set the application website, use [property`AboutWindow`:py:data::website<Adw.AboutWindow.props.website>]. To add extra links below the website, use [method`AboutWindow`.add_link].

If the Details page doesn’t have any other content besides website, the website will be displayed on the main page instead.

Troubleshooting

AdwAboutWindow displays the following two links on the main page:

  • Support Questions, set with the [property`AboutWindow`:py:data::support-url<Adw.AboutWindow.props.support_url>] property,

  • Report an Issue, set with the [property`AboutWindow`:py:data::issue-url<Adw.AboutWindow.props.issue_url>] property.

Additionally, applications can provide debugging information. It will be shown separately on the Troubleshooting page. Use the [property`AboutWindow`:py:data::debug-info<Adw.AboutWindow.props.debug_info>] property to specify it.

It’s intended to be attached to issue reports when reporting issues against the application. As such, it cannot contain markup or links.

AdwAboutWindow provides a quick way to save debug information to a file. When saving, [property`AboutWindow`:py:data::debug-info-filename<Adw.AboutWindow.props.debug_info_filename>] would be used as the suggested filename.

Credits and Acknowledgements

The Credits page has the following default sections:

  • Developers, set with the [property`AboutWindow`:py:data::developers<Adw.AboutWindow.props.developers>] property,

  • Designers, set with the [property`AboutWindow`:py:data::designers<Adw.AboutWindow.props.designers>] property,

  • Artists, set with the [property`AboutWindow`:py:data::artists<Adw.AboutWindow.props.artists>] property,

  • Documenters, set with the [property`AboutWindow`:py:data::documenters<Adw.AboutWindow.props.documenters>] property,

  • Translators, set with the [property`AboutWindow`:py:data::translator-credits<Adw.AboutWindow.props.translator_credits>] property.

When setting translator credits, use the strings "translator-credits" or "translator_credits" and mark them as translatable.

The default sections that don’t contain any names won’t be displayed.

The Credits page can also contain an arbitrary number of extra sections below the default ones. Use [method`AboutWindow`.add_credit_section] to add them.

The Acknowledgements page can be used to acknowledge additional people and organizations for their non-development contributions. Use [method`AboutWindow`.add_acknowledgement_section] to add sections to it. For example, it can be used to list backers in a crowdfunded project or to give special thanks.

Each of the people or organizations can have an email address or a website specified. To add a email address, use a string like Edgar Allan Poe <edgar@poe.com>. To specify a website with a title, use a string like The GNOME Project https://www.gnome.org :

<picture> <source srcset=”about-window-credits-dark.png” media=”(prefers-color-scheme: dark)”> <img src=”about-window-credits.png” alt=”about-window-credits”> </picture>

Legal

The Legal page displays the copyright and licensing information for the application and other modules.

The copyright string is set with the [property`AboutWindow`:py:data::copyright<Adw.AboutWindow.props.copyright>] property and should be a short string of one or two lines, for example: © 2022 Example.

Licensing information can be quickly set from a list of known licenses with the [property`AboutWindow`:py:data::license-type<Adw.AboutWindow.props.license_type>] property. If the application’s license is not in the list, [property`AboutWindow`:py:data::license<Adw.AboutWindow.props.license>] can be used instead.

To add information about other modules, such as application dependencies or data, use [method`AboutWindow`.add_legal_section].

Constructing

To make constructing an AdwAboutWindow as convenient as possible, you can use the function [func`show_about_window`] which constructs and shows a window.

```c static void show_about (Gtk.Application *app) { const str *developers[] = { “Angela Avery”, None };

const str *designers[] = { “GNOME Design Team”, None };

adw_show_about_window (Gtk.Application.get_active_window (app), “application-name”, _(“Example”), “application-icon”, “org.example.App”, “version”, “1.2.3”, “copyright”, “© 2022 Angela Avery”, “issue-url”, “https://gitlab.gnome.org/example/example/-/issues/new”, “license-type”, Gtk.License.GPL_3_0, “developers”, developers, “designers”, designers, “translator-credits”, _(“translator-credits”), None); } ```

CSS nodes

AdwAboutWindow has a main CSS node with the name window and the style class .about.

classmethod new_from_appdata(resource_path, release_notes_version)¶
Parameters:

  • resource_path (str) – The resource to use

  • release_notes_version (str or None) – The version to retrieve release notes for

Returns:

the newly created AdwAboutWindow

Return type:

Gtk.Widget

Creates a new AdwAboutWindow using AppStream metadata.

This automatically sets the following properties with the following AppStream values:

  • [property`AboutWindow`:py:data::application-icon<Adw.AboutWindow.props.application_icon>] is set from the <id>

  • [property`AboutWindow`:py:data::application-name<Adw.AboutWindow.props.application_name>] is set from the <name>

  • [property`AboutWindow`:py:data::developer-name<Adw.AboutWindow.props.developer_name>] is set from the <developer_name>

  • [property`AboutWindow`:py:data::version<Adw.AboutWindow.props.version>] is set from the version of the latest release

  • [property`AboutWindow`:py:data::website<Adw.AboutWindow.props.website>] is set from the <url type="homepage">

  • [property`AboutWindow`:py:data::support-url<Adw.AboutWindow.props.support_url>] is set from the <url type="help">

  • [property`AboutWindow`:py:data::issue-url<Adw.AboutWindow.props.issue_url>] is set from the <url type="bugtracker">

  • [property`AboutWindow`:py:data::license-type<Adw.AboutWindow.props.license_type>] is set from the <project_license> If the license type retrieved from AppStream is not listed in [enum`Gtk`.License], it will be set to GTK_LICENCE_CUSTOM.

If release_notes_version is not NULL, [property`AboutWindow`:py:data::release-notes-version<Adw.AboutWindow.props.release_notes_version>] is set to match it, while [property`AboutWindow`:py:data::release-notes<Adw.AboutWindow.props.release_notes>] is set from the AppStream release description for that version.

add_legal_section(title, copyright, license_type, license)¶
Parameters:

  • title (str) – the name of the section

  • copyright (str or None) – a copyright string

  • license_type (Gtk.License) – the type of license

  • license (str or None) – custom license information

Adds an extra section to the Legal page.

Extra sections will be displayed below the application’s own information.

The parameters copyright, license_type and license will be used to present the it the same way as [property`AboutWindow`:py:data::copyright<Adw.AboutWindow.props.copyright>], [property`AboutWindow`:py:data::license-type<Adw.AboutWindow.props.license_type>] and [property`AboutWindow`:py:data::license<Adw.AboutWindow.props.license>] are for the application’s own information.

See those properties for more details.

This can be useful to attribute the application dependencies or data.

Examples:

```c Adw.AboutWindow.add_legal_section (ADW_ABOUT_WINDOW (about), _(“Copyright and a known license”), “© 2022 Example”, Gtk.License.LGPL_2_1, None);

Adw.AboutWindow.add_legal_section (ADW_ABOUT_WINDOW (about), _(“Copyright and custom license”), “© 2022 Example”, Gtk.License.CUSTOM, “Custom license text”);

Adw.AboutWindow.add_legal_section (ADW_ABOUT_WINDOW (about), _(“Copyright only”), “© 2022 Example”, Gtk.License.UNKNOWN, None);

Adw.AboutWindow.add_legal_section (ADW_ABOUT_WINDOW (about), _(“Custom license only”), None, Gtk.License.CUSTOM, “Something completely custom here.”); ```

set_release_notes(release_notes)¶
Parameters:

release_notes (str) – the release notes

Sets the release notes for self.

Release notes are displayed on the the What’s New page.

Release notes are formatted the same way as AppStream descriptions.

The supported formatting options are:

  • Paragraph (<p>)

  • Ordered list (<ol>), with list items (<li>)

  • Unordered list (<ul>), with list items (<li>)

Within paragraphs and list items, emphasis (<em>) and inline code (<code>) text styles are supported. The emphasis is rendered in italic, while inline code is shown in a monospaced font.
