get_master_root()¶
Returns:the effective root for all locations, or None on error. The root is owned by self and should not be freed. It can be referenced using GObject.Object.ref().
Return type:Gio.File

Returns the Gio.File that represents the master root location for all indexing locations. For example, if file:///etc is an indexed path and so was file:///home/user, the master root is file:///. Only one scheme per self can be used, so you can not mix http and file roots in self.

The return value should NEVER be None. In cases where no root is given, we fallback to file:///.

Roots explained:

  • master root = top most level root node, e.g. file:///
  • config root = a root node from Gio.Settings, e.g. file:///home/martyn/Documents
  • root = ANY root, normally config root, but it can also apply to roots added for devices, which technically are not a config root or a master root.

New in version 1.2.

Returns:

Return type:

bool

Signals either TrackerMiner.IndexingTree ::directory-updated or TrackerMiner.IndexingTree ::child-updated on the given file and returns True. If file is not indexed according to the TrackerMiner.IndexingTree, False is returned.

If recursive is True, TrackerMiner.IndexingTree ::directory-updated will be emitted on the indexing roots that are contained in file.

set_default_policy(filter, policy)¶
Parameters:
  • filter (TrackerMiner.FilterType) – a TrackerMiner.FilterType
  • policy (TrackerMiner.FilterPolicy) – a TrackerMiner.FilterPolicy

Set the default policy (to allow or deny) for content in self based on the type - in this case filter. Here, filter is a file or directory and there are some other options too.

For example, you can (by default), disable indexing all directories using this function.

New in version 0.18.

set_filter_hidden(filter_hidden)¶
Parameters:filter_hidden (bool) – a boolean

When indexing content, sometimes it is preferable to ignore hidden content, for example, files prefixed with “.”. This is common for files in a home directory which are usually config files.

Sets the indexing policy for self with hidden files and content. To ignore hidden files, filter_hidden should be True, otherwise False.

New in version 0.18.

RUN_LAST
Parameters:

  • root (Gio.File) – the root of this child
  • child (Gio.File) – the updated child

The ::child-updated signal may be emitted to notify about possible changes on children of a root.

TrackerMiner.IndexingTree does not emit those by itself, those may be triggered through TrackerMiner.IndexingTree.notify_update().

New in version 1.10.
