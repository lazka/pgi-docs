pause()¶

Asks self to pause. This call may be called multiple times, but TrackerMiner.Miner ::paused will only be emitted the first time. The same number of TrackerMiner.Miner.resume() calls are expected in order to resume operations.

resume()¶
Returns:True if the miner resumed its operations.
Return type:bool

Asks the miner to resume processing. This needs to be called as many times as TrackerMiner.Miner.pause() calls were done previously. This function will return True when the miner is actually resumed.

TrackerMiner.Miner.signals.progress(miner, status, progress, remaining_time)¶
Signal Name:

progress
Flags:

RUN_LAST
Parameters:
  • miner (TrackerMiner.Miner) – The object which received the signal
  • status (str) – miner status
  • progress (float) – a float indicating miner progress, from 0 to 1.
  • remaining_time (int) – a int indicating the reamaining processing time, in seconds.

the ::progress signal will be emitted by TrackerMiner.Miner implementations to indicate progress about the data mining process. status will contain a translated string with the current miner status and progress will indicate how much has been processed so far. remaining_time will give the number expected of seconds to finish processing, 0 if the value cannot be estimated, and -1 if its not applicable.

New in version 0.12.
