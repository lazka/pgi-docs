WebKit2.BackForwardListClass

List of visited pages.

WebKit2.BackForwardList maintains a list of visited pages used to navigate to recent pages. Items are inserted in the list in the order they are visited.

WebKit2.BackForwardList also maintains the notion of the current item (which is always at index 0), the preceding item (which is at index -1), and the following item (which is at index 1). Methods WebKit2.WebView.go_back() and WebKit2.WebView.go_forward() move the current item backward or forward by one. Method WebKit2.WebView.go_to_back_forward_list_item() sets the current item to the specified item. All other methods returning WebKit2.BackForwardListItem s do not change the value of the current item, they just return the requested item or items.

Signal Details¶

WebKit2.BackForwardList.signals.changed(back_forward_list, item_added, items_removed)¶
Signal Name

changed

Flags

RUN_LAST

Parameters

  • back_forward_list (WebKit2.BackForwardList) – The object which received the signal

  • item_added (WebKit2.BackForwardListItem or None) – the WebKit2.BackForwardListItem added or None

  • items_removed (object or None) – a GLib.List of WebKit2.BackForwardListItem s

This signal is emitted when back_forward_list changes. This happens when the current item is updated, a new item is added or one or more items are removed. Note that both item_added and items_removed can None when only the current item is updated. Items are only removed when the list is cleared or the maximum items limit is reached.
