get_ims_services_enabled_setting(unused, timeout, cancellable, callback, *user_data)¶
Parameters:

  • unused (object or None) – None. This message doesn’t have any input bundle.

  • timeout (int) – maximum time to wait for the method to complete, in seconds.

  • cancellable (Gio.Cancellable or None) – a Gio.Cancellable or None.

  • callback (Gio.AsyncReadyCallback or None) – a Gio.AsyncReadyCallback to call when the request is satisfied.

  • user_data (object or None) – user data to pass to callback.

Asynchronously sends a Get IMS Services Enabled Setting request to the device.

When the operation is finished, callback will be invoked in the thread-default main loop of the thread you are calling this method from.

You can then call Qmi.ClientIms.get_ims_services_enabled_setting_finish() to get the result of the operation.

New in version 1.34.
