The GtkPicture widget displays a GdkPaintable.

An example Gtk.Picture

Many convenience functions are provided to make pictures simple to use. For example, if you want to load an image from a file, and then display it, thereâs a convenience function to do this:

``c GtkWidget *widget = gtk_picture_new_for_filename (âmyfile.pngâ); ``

If the file isnât loaded successfully, the picture will contain a âbroken imageâ icon similar to that used in many web browsers. If you want to handle errors in loading the file yourself, for example by displaying an error message, then load the image with [ctor`Gdk`.Texture.new_from_file], then create the GtkPicture with [ctor`Gtk`.Picture.new_for_paintable].

Sometimes an application will want to avoid depending on external data files, such as image files. See the documentation of GResource for details. In this case, [ctor`Gtk`.Picture.new_for_resource] and [method`Gtk`.Picture.set_resource] should be used.

GtkPicture displays an image at its natural size. See [class`Gtk`.Image] if you want to display a fixed-size image, such as an icon.

Sizing the paintable

You can influence how the paintable is displayed inside the GtkPicture by changing [property`Gtk`.Picture:content-fit]. See [enum`Gtk`.ContentFit] for details. [property`Gtk`.Picture:can-shrink] can be unset to make sure that paintables are never made smaller than their ideal size - but be careful if you do not know the size of the paintable in use (like when displaying user-loaded images). This can easily cause the picture to grow larger than the screen. And [property`Gtk`.Widget:halign] and [property`Gtk`.Widget:valign] can be used to make sure the paintable doesnât fill all available space but is instead displayed at its original size.

CSS nodes

GtkPicture has a single CSS node with the name picture.

Accessibility

GtkPicture uses the GTK_ACCESSIBLE_ROLE_IMG role.

set_can_shrink(can_shrink)[source]Â¶
Parameters

can_shrink (bool) â if self can be made smaller than its contents

If set to True, the self can be made smaller than its contents.

The contents will then be scaled down when rendering.

If you want to still force a minimum size manually, consider using [method`Gtk`.Widget.set_size_request].

Also of note is that a similar function for growing does not exist because the grow behavior can be controlled via [method`Gtk`.Widget.set_halign] and [method`Gtk`.Widget.set_valign].

keep_aspect_ratio (bool) â whether to keep aspect ratio

If set to True, the self will render its contents according to their aspect ratio.

That means that empty space may show up at the top/bottom or left/right of self.

If set to False or if the contents provide no aspect ratio, the contents will be stretched over the pictureâs whole area.
