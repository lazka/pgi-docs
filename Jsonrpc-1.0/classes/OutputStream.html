The Jsonrpc.OutputStream is resonsible for serializing messages onto the underlying stream.

Optionally, if Jsonrpc.OutputStream.set_use_gvariant() has been called, the messages will be encoded directly in the GLib.Variant format instead of JSON along with setting a âContent-Typeâ header to âapplication/gvariantâ. This is useful for situations where you control both sides of the RPC server using jsonrpc-glib as you can reduce the overhead of parsing JSON nodes due to GLib.Variant not requiring parsing or allocation overhead to the same degree as JSON.

For example, if you need a large message, which is encoded in JSON, you need to decode the entire message up front which avoids performing lazy operations. When using GLib.Variant encoding, you have a single allocation created for the GLib.Variant which means you reduce the memory pressure caused by lots of small allocations.
