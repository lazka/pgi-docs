Details¶

EDataCal.CalBackendCustomOpFunc(cal_backend, user_data, cancellable)¶
Parameters:
  • cal_backend (EDataCal.CalBackend) – an EDataCal.CalBackend
  • user_data (object or None) – a function user data, as provided to EDataCal.CalBackend.schedule_custom_operation()
  • cancellable (Gio.Cancellable or None) – an optional Gio.Cancellable, as provided to EDataCal.CalBackend.schedule_custom_operation()

A callback prototype being called in a dedicated thread, scheduled by EDataCal.CalBackend.schedule_custom_operation().

New in version 3.26.

EDataCal.CalBackendForeachViewFunc(backend, view, *user_data)¶
Parameters:
  • backend (EDataCal.CalBackend) – an EDataCal.CalBackend
  • view (EDataCal.DataCalView) – an EDataCal.DataCalView
  • user_data (object or None) – user data for the function
Returns:

True, to continue, False to stop further processing.
Return type:

bool

Callback function used by EDataCal.CalBackend.foreach_view().

New in version 3.34.

EDataCal.CalCacheSearchFunc(cal_cache, uid, rid, revision, object, extra, custom_flags, offline_state, *user_data)¶
Parameters:
  • cal_cache (EDataCal.CalCache) – an EDataCal.CalCache
  • uid (str) – a unique object identifier
  • rid (str or None) – an optional Recurrence-ID of the object
  • revision (str) – the object revision
  • object (str) – the object itself
  • extra (str) – extra data stored with the object
  • custom_flags (int) – object’s custom flags
  • offline_state (EBackend.OfflineState) – object’s offline state, one of EBackend.OfflineState
  • user_data (object or None) – user data, as used in EDataCal.CalCache.search_with_callback()
Returns:

True to continue, False to stop walk through.
Return type:

bool

A callback called for each object row when using EDataCal.CalCache.search_with_callback() function.

New in version 3.26.
