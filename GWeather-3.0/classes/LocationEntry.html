A subclass of Gtk.SearchEntry that provides autocompletion on GWeather.Location s

classmethod new(top)¶
Parameters:top (GWeather.Location) – the top-level location for the entry.
Returns:the new GWeather.LocationEntry
Return type:Gtk.Widget

Creates a new GWeather.LocationEntry.

top will normally be the location returned from GWeather.Location.get_world(), but you can create an entry that only accepts a smaller set of locations if you want.

get_location()¶
Returns:the selected location (which you must unref when you are done with it), or None if no location is selected.
Return type:GWeather.Location or None

Gets the location that was set by a previous call to GWeather.LocationEntry.set_location() or was selected by the user.

has_custom_text()¶
Returns:True if self’s text was modified by the user, or False if it’s set to the default text of a location.
Return type:bool

Checks whether or not self’s text has been modified by the user. Note that this does not mean that no location is associated with self. GWeather.LocationEntry.get_location() should be used for this.

set_city(city_name, code)¶
Parameters:
  • city_name (str or None) – the city name, or None
  • code (str) – the METAR station code
Returns:

True if self’s location could be set to a matching city, False otherwise.
Return type:

bool

Sets self’s location to a city with the given code, and given city_name, if non-None. If there is no matching city, sets self’s location to None.

set_location(loc)¶
Parameters:loc (GWeather.Location or None) – a GWeather.Location in self, or None to clear self

Sets self’s location to loc, and updates the text of the entry accordingly. Note that if the database contains a location that compares equal to loc, that will be chosen in place of loc.
