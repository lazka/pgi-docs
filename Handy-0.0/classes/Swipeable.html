Signal Details¶

Handy.Swipeable.signals.begin_swipe(swipeable, direction)¶
Signal Name:

begin-swipe
Flags:

RUN_FIRST
Parameters:
  • swipeable (Handy.Swipeable) – The object which received the signal
  • direction (int) – The direction of the swipe, can be 1 or -1

This signal is emitted when a possible swipe is detected. This is used by Handy.SwipeGroup, applications should not connect to it. The direction value can be used to restrict the swipe to a certain direction.

New in version 0.0.12.

Handy.Swipeable.signals.end_swipe(swipeable, duration, to)¶
Signal Name:

end-swipe
Flags:

RUN_FIRST
Parameters:
  • swipeable (Handy.Swipeable) – The object which received the signal
  • duration (int) – Snap-back animation duration in milliseconds
  • to (float) – The progress value to animate to

This signal is emitted as soon as the gesture has stopped. This is used by Handy.SwipeGroup, applications should not connect to it.

New in version 0.0.12.

Handy.Swipeable.signals.switch_child(swipeable, index, duration)¶
Signal Name:

switch-child
Flags:

RUN_FIRST
Parameters:
  • swipeable (Handy.Swipeable) – The object which received the signal
  • index (int) – the index of the child to switch to
  • duration (int) – Animation duration in milliseconds

This signal should be emitted when the widget’s visible child is changed.

duration can be 0 if the child is switched without animation.

New in version 0.0.12.

Handy.Swipeable.signals.update_swipe(swipeable, value)¶
Signal Name:

update-swipe
Flags:

RUN_FIRST
Parameters:
  • swipeable (Handy.Swipeable) – The object which received the signal
  • value (float) – The current animation progress value

This signal is emitted every time the progress value changes. This is used by Handy.SwipeGroup, applications should not connect to it.

New in version 0.0.12.
