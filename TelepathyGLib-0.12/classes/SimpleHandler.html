classmethod new(dbus, bypass_approval, requests, name, uniquify, callback, *user_data)¶
Parameters:

  • dbus (TelepathyGLib.DBusDaemon) – a TelepathyGLib.DBusDaemon object, may not be None

  • bypass_approval (bool) – the value of the Handler.BypassApproval D-Bus property (see TelepathyGLib.BaseClient.set_handler_bypass_approval() for details)

  • requests (bool) – whether this handler should implement Requests (see TelepathyGLib.BaseClient.set_handler_request_notification() for details)

  • name (str) – the name of the Handler (see TelepathyGLib.BaseClient :name for details)

  • uniquify (bool) – the value of the TelepathyGLib.BaseClient :uniquify-name property

  • callback (TelepathyGLib.SimpleHandlerHandleChannelsImpl) – the function called when HandleChannels is called

  • user_data (object or None) – arbitrary user-supplied data passed to callback

Returns:

a new TelepathyGLib.SimpleHandler

Return type:

TelepathyGLib.SimpleHandler

Convenient function to create a new TelepathyGLib.SimpleHandler instance.

If dbus is not the result of TelepathyGLib.DBusDaemon.dup(), you should call TelepathyGLib.SimpleHandler.new_with_am() instead, so that TelepathyGLib.Account, TelepathyGLib.Connection and TelepathyGLib.Contact instances can be shared between modules.

New in version 0.11.6.

Deprecated since version ???: New code should use TelepathyGLib.SimpleHandler.new_with_am() instead.
